## 1. 项目概述

| 项目 | 说明 |
|------|------|
| **产品名称** | 纪念日小站（只为你而做） |
| **产品类型** | 静态多页 H5，情侣纪念/礼物向 |
| **部署方式** | GitHub Pages，单链接访问，手机为主 |
| **目标** | 一个链接打开首页，从首页可进入多个子页面；实用、可长期使用，风格统一。 |

---

## 2. 主题与视觉规范

**整体主题：冰蓝色（全站统一）**

| 项目 | 规范 |
|------|------|
| **背景** | 深冰蓝渐变：`#0a1929` → `#0d2847` → `#132f4c` → `#0a1929` |
| **主色/强调色** | 冰蓝：`#81d4fa`、`#4fc3f7`、`#29b6f6` |
| **卡片/边框** | `rgba(129,212,250,0.08)` 背景，`rgba(129,212,250,0.25~0.3)` 边框 |
| **文字** | 主文案白色/近白，辅助文案 `rgba(255,255,255,0.5~0.7)` |
| **theme-color** | `#0d2137`（浏览器栏等） |
| **字体** | -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "PingFang SC", "Microsoft YaHei", sans-serif |
| **布局** | 内容区最大宽度 420px 居中，左右留白 20px，支持安全区域（刘海屏、横条）；移动端优先。 |

所有新增页面须与上述主题一致，不得使用粉色/红色等冲突色系。

## 4. 数据与配置

### 4.1 纪念日配置（当前在 `love-days.html` 内 CONFIG）

| 配置项 | 含义 | 当前值 |
|--------|------|--------|
| `startDate` | 在一起的日子 | `2023-03-22` |
| `nextDate` | 下一个纪念日日期 | `2025-02-14` |
| `nextLabel` | 纪念日显示名称 | `情人节` |
| `nickname` | 对方昵称（预留） | `你` |

修改纪念日或昵称时，仅需改上述 CONFIG，无需改需求文档。

### 4.2 图片规范

- **存放位置**：项目内 `images/`（或 `photos/`）文件夹，与 HTML 同级。
- **引用方式**：相对路径，如 `images/photo1.jpg`。
- **建议**：英文文件名、适当压缩、单张约 1～2MB 内，便于 GitHub 与手机加载。
- 图片与 HTML 一并上传至同一 GitHub 仓库即可，无需外链图床。

## 5. 技术约束

| 项目 | 约束 |
|------|------|
| **技术栈** | 纯静态：HTML + CSS + JavaScript，无后端、无构建工具。 |
| **部署** | GitHub Pages，仓库公开时 Pages 可用；私有仓库免费账号不可用 Pages。 |
| **兼容** | 现代浏览器、移动端优先；使用 safe-area-inset、100dvh 等适配全面屏。 |
| **数据持久化** | 仅允许使用前端能力（如 localStorage），用于心愿清单等勾选状态。 |
| **新增页面** | 须含「← 返回首页」链回 `index.html`，且视觉风格符合第 2 节冰蓝色主题。 |
